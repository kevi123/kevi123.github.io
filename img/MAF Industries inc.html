Multi-Threading
Can you describe a project using Multi-Threading you have done?
I have not been fortunate enough to create a software project that has taken advantage multi-thread execution. However, I have done work to simulated multi-thread execution and I have familiarity with how it works at the low level of the computer architecture. I am aware of the processes that are occurring in the assembly instructions and in the registers. I do have an understanding of the the computing pipelines, clock cycles per given instructions, clock speeds, dependencies, stalls and latency of instructions. I have been exposed to bubbles (stalls) and the impact on the software wait times and clock cycle times. I am familiar with the stages of pipelining (Fetching, Decoding, Execution, Memory access and Write to Register) and would like to apply this understanding to practical projects.
What type of project was it?
What was the biggest challenge you faced?
The biggest challenge is grasping the the process of multi-thread execution as a whole. The biggest challenge I faced was optimizing the assembly instructions in order to not cause any stalls in the program.
How did you solve your prevoulsy described challenge?
Fortunately, we were not alone in learning these techniques to optimize the code. Working together with my peers we were able to find the best ways to optimize the code. We were able to work together to order instructions where we would have the least stalls and dependencies occurring. It was rewarding to truly be able to understand why we should avoid Read After Write among other hazardous situations.
Computer Vision
Can you describe your most significant Computer Vision project?
The most significant computer vision project I worked with was an autonomous turret gun. In this project we used a raspberry pi unit to control the project. On the raspberry pi we were able to utilize the General Purpose Input Output pins to attach various sensors for data collection. We used modified push buttons to detect when the cannon was positioned correctly. OpenCv software, PiCam and python programming was used for the object detection. An air valve was used to shoot the safe projectiles(ping pong balls) once the person was detected. We used stepper motors for the movement of the cannon. We also re used parts from used electronics in order to complete the project, for instance a computer printer was dismantled in order to have stepper motors available for this project. A lot of soldering was conducted in order to secure much of the wiring.
What type of project was it?
What was the biggest challenge you faced?
The biggest challenge was having the hardware parts readily available.
How did you solve your previously described challenge?
As mentioned, many times we would not have the correct hardware for the application. Many times we have to get creative. During this project we were in need of stepper motors that were strong enough to shift the cannon. While the items could have been ordered on line, we did not have the luxury of time to wait. We would get savvy and re-purpose components of electronics that we did have available. For example we re used old Christmas lights for LED effects, we removed servo motors from remote control cars, stepper motors from printers, splice Ethernet cables for wires. After all this I feel that we were able to understand how many electronics operate, and as a team we could find ways to complete the tasks at hand with some ingenuity and the tools at hand.
